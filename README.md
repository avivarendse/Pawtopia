# Pawtopia

## Startup Description

Pawtopia is a pet care startup designed to connect pet owners with reliable daycare, boarding, grooming, and pet care services. The platform provides a safe, convenient, and trustworthy solution for pet owners who need quality care for their pets when they are busy, travelling, or unavailable.

## Problem Statement

Many pet owners struggle to find reliable and trustworthy pet care services when they are away from home due to work, travel, or other commitments. Pawtopia solves this problem by offering a centralised and professional platform where pet owners can access safe and dependable pet care services.

## Mission Statement

To provide safe, loving, and dependable pet care services that give pet owners peace of mind while ensuring pets receive high-quality care and attention.

## Target Audience

- Pet owners  
- Working professionals  
- Frequent travellers  
- Families with pets  
- Individuals needing short-term or long-term pet care services  

## Unique Selling Proposition (USP)

Pawtopia offers a complete pet care solution including daycare, grooming, boarding, and personalised care services through a user-friendly and modern platform focused on trust, safety, and convenience.

## Features

- Responsive multi-page website  
- Navigation bar with mobile menu  
- Hero section with call-to-action buttons  
- Dog and Cat owner content switching feature  
- Services showcase pages  
- Contact form for enquiries  
- Modern UI design with consistent branding  
- Fully responsive layout for mobile and desktop  
- Interactive JavaScript functionality  

## HTML Concepts Used

- HTML5 document structure  
- Semantic elements (header, main, section, footer)  
- Headings and paragraphs  
- Lists and links  
- Images with alt attributes  
- Forms, inputs, and labels  
- Buttons  
- Meta tags and proper page structure  
- Comments for code clarity  

## CSS Concepts Used

- External stylesheets  
- Class and ID selectors  
- Flexbox layout system  
- Grid layout system  
- Box model (margin, padding, borders)  
- Typography and Google Fonts  
- Colour schemes and branding consistency  
- Hover effects and transitions  
- Responsive design with media queries  
- Mobile-first design principles  

## JavaScript Concepts Used

- Variables and data types  
- Functions  
- Event listeners  
- DOM manipulation  
- Conditional statements  
- Arrays and objects  
- Dynamic content updates  
- Interactive navigation menu  
- UI state changes (Dog/Cat toggle feature)  

## SDLC Process

### Planning
The team defined the startup idea, assigned roles, and outlined the website structure.

### Analysis
User needs were identified, focusing on pet owners requiring reliable pet care services.

### Design
Figma wireframes and prototypes were created to plan layout, branding, and user experience.

### Development
The website was built using HTML, CSS, and JavaScript with GitHub for version control.

### Testing
The team tested responsiveness, navigation, layout consistency, and JavaScript functionality.

### Deployment
The project was prepared for deployment as a live website.

## Challenges Faced

- Coordinating work across multiple team members  
- Merging contributions using GitHub  
- Maintaining consistent design across pages  
- Ensuring responsive design across devices  

## Lessons Learned

- Importance of planning before coding  
- Real-world use of Git and GitHub collaboration  
- How SDLC improves project structure and teamwork  
- How JavaScript adds interactivity to websites  
- Importance of communication in team development  

## Team Reflection

We created Pawtopia, a pet care startup focused on making pet services more accessible and reliable for pet owners.

The idea was chosen because many pet owners need trustworthy care solutions when they are unavailable.

The project solved the problem of finding safe, convenient, and reliable pet care services.

Aviv worked as the project coordinator, handling planning, documentation, GitHub management, and deployment. Khanya developed the homepage. Stephan built the services page. Michaela and Phelisa worked on the About and Contact pages. Avela handled JavaScript functionality and testing. All members contributed to styling, responsiveness, and integration.

The team faced challenges with coordination, merging code, and maintaining consistency across pages. These were solved through communication, GitHub collaboration, and regular team reviews.

We learned how real development teams work using SDLC, GitHub workflows, and collaborative coding practices. We also improved our understanding of JavaScript and responsive design.

We are most proud of successfully building a fully functional startup website as a team.

If we could improve anything, we would add more advanced features such as online booking, user accounts, and database integration.
